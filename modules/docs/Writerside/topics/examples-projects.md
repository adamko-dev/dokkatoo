# Example Projects

Dokkatoo GitHub has some examples of how to use Dokkatoo.

These are primarily comparative examples with existing Dokka examples.

### Custom Format Example Project

This example demonstrates how to override `.css` styles and add custom images as assets, allowing
you to change the logo used in the header.

[https://github.com/adamko-dev/dokkatoo/tree/main/examples/custom-format-example/dokkatoo](https://github.com/adamko-dev/dokkatoo/tree/main/examples/custom-format-example/dokkatoo)

### Gradle Example Project

This example demonstrates how to apply Dokka in a simple single-project Gradle build, as well as how
to configure it.

[https://github.com/adamko-dev/dokkatoo/tree/main/examples/gradle-example/dokkatoo](https://github.com/adamko-dev/dokkatoo/tree/main/examples/gradle-example/dokkatoo)

### Kotlin-as-Java Example Project

This example demonstrates how you can apply a Dokka plugin in a simple Gradle project.

In particular, it applies
[Kotlin as Java](https://github.com/Kotlin/dokka/tree/v1.9.20/dokka-subprojects/plugin-kotlin-as-java)
Dokka plugin that renders all
Kotlin signatures as Java signatures.

[https://github.com/adamko-dev/dokkatoo/tree/main/examples/kotlin-as-java-example/dokkatoo](https://github.com/adamko-dev/dokkatoo/tree/main/examples/kotlin-as-java-example/dokkatoo)

### Kotlin Multiplatform Example Project

This example demonstrates Dokka's configuration and output for a simple
[Kotlin Multiplatform](https://kotlinlang.org/docs/multiplatform.html) project.

[https://github.com/adamko-dev/dokkatoo/tree/main/examples/multiplatform-example/dokkatoo](https://github.com/adamko-dev/dokkatoo/tree/main/examples/multiplatform-example/dokkatoo)

### Library Publishing Example Project

This example demonstrates how you can integrate Dokka into the publishing process of your library,
adding documentation generated by Dokka as artifacts.

This is useful because some repositories, like Maven Central, require documentation (`javadoc.jar`)
to be published alongside library artifacts.

> No Dokkatoo example yet - contributions welcome!
> {style="warning"}

[https://github.com/adamko-dev/dokkatoo/tree/main/examples/library-publishing-example/dokkatoo](https://github.com/adamko-dev/dokkatoo/tree/main/examples/library-publishing-example/dokkatoo)

### Multimodule Example Project

[https://github.com/adamko-dev/dokkatoo/tree/main/examples/multiplatform-example/dokkatoo](https://github.com/adamko-dev/dokkatoo/tree/main/examples/multiplatform-example/dokkatoo)

### Versioning Example Project

This example demonstrates configuration of Dokka's
[versioning plugin](https://github.com/Kotlin/dokka/tree/v1.9.20/dokka-subprojects/plugin-versioning),
which allows readers to navigate through different versions of your API reference documentation.

[https://github.com/adamko-dev/dokkatoo/tree/main/examples/versioning-multimodule-example/dokkatoo](https://github.com/adamko-dev/dokkatoo/tree/main/examples/versioning-multimodule-example/dokkatoo)

## Contributors details

If you want to contribute an example, then it would be welcomed and we would be grateful!

Here are some details about the example projects set up that are useful for contributors.

### Set up

The Dokka examples are synced automatically from the Dokka source code.

### Tests

The projects are tested in the
[`:modules:dokkatoo-plugin-integration-tests`](https://github.com/adamko-dev/dokkatoo/tree/main/modules/dokkatoo-plugin-integration-tests)
project.

The Dokka Publications generated by the Dokka examples are compared against the Dokka
Publications generated by the Dokkatoo projects.
